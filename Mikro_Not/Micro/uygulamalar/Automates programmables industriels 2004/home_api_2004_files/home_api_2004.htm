LES AUTOMATES PROGRAMMABLES INDUSTRIELS 2004 D'ULYSSE
A BASE DE PIC16F873, 16F874, 16F876 OU 16F877


Cette page permet de télécharger la description, les fichiers et le logiciel afin de realiser vous même et d'utiliser mes API 2004.



Ces API sont une evolution de mes precedents automates a base de PIC16F84.

Ils sont plus performants et simples d'utilisation.

La documentation décrit en detail cette realisation.

Contactez moi pour toute question.

Bien entendu, cette réalisation est entièrement gratuite.

Dernières nouvelles :

  • 1 Aout 2005Attention, n'utilisez pas les versions A des PICs (ex 16f876A). En effet, Microchip a changé la procedure programmation de la FLASH par le firmware. Vous ne pourrez donc pas télécharger le grapcet dans la flash. si vous souhaitez utiliser une version A, la solution a ce probleme est d'utiliser icprog + JDM pour flasher le fichier .hex disponible en fin de compilation. Ce fichier contient le firmware plus le graphcet.
  • 24 mai 2005:La societe Modtronix m'a propose de diffuser un Kit de l'API 10E/8S basé sur une de leur carte. J'ai accepté gratuitement afin d'aider ceux qui hesite a se lancer dans une réalisation complète. Mais il faudra tout de meme réaliser une carte fille non incluse pour la partie interface (E/S, relais). Ce kit comprend une carte avec un PIC programmé, un cable et un CD. En toute mpartialite, a vous de voir, si cela vaut le coup ici.
  • 26 avril mars 2005: Correction d'un probleme de progapi (dans certaines conditions la compilation d'une etape sans aucune action echouée). Pour mettre a jour, Téléchargez l'archive et remplacer juste progapi_2004.exe par prograpi_2004_101.exe (v1.01). Merci a Laurent pour avoir trouvé ce probleme.
  • 31 mars 2005: Correction d'un pb de comparaison des variables 8 bits (can ...). Merci à Gilbert pour l'identification de ce probleme. Téléchargez le nouveau zip afin de mettre à jour les fichiers api2004a.hex et api2004a.def. Il faut aussi reflasher le UC avec le nouveau api2004a.hex
  • 07 jan 2005: Correction d'un pb sur le portD pour l'automate 15E/14S. Merci à Jean Pierre pour l'identification et la correction de ce probleme. Téléchargez le nouveau zip afin de mettre à jour les fichiers api2004a.hex et api2004a.def. Il faut aussi reflasher le UC avec le nouveau api2004a.hex
  • 22 nov 2004: Correction d'un pb firmware pour l'automate 15E/14S. Téléchargez le nouveau zip afin de mettre à jour les fichiers api2004a.hex et api2004a.def. Il faut aussi reflasher le UC avec api2004a.hex
